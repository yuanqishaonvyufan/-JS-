百度翻译网页接口逆向逻辑

一、目标
不使用浏览器自动化，直接用 Python 调用百度翻译网页接口，实现批量翻译。

二、网站的阻碍
1. 翻译接口不是公开 API。
2. 请求需要动态 Acs-Token，用于检测浏览器环境和拦截异常请求。
3. 翻译结果通过 SSE 流式返回，不能直接当普通 JSON 读取。

三、解决办法
1. 抓包定位真实接口：POST /ait/text/translate。
2. 还原 query、from、to、时间戳等 JSON 请求参数。
3. 定位 Paris/ACS SDK → getSign() → Acs-Token 的调用链。
4. 使用自己成功抓包得到的短期 Acs-Token、Cookie 和对应请求头重放请求。
5. 解析 SSE 中的 Translating 事件，拼接最终译文。

注意：本项目没有伪造或完整破解 ACS 内部风控算法，只还原了业务请求流程，并复用合法抓包得到的短期 Token。

四、最终结果
Python 可以在 Token 有效期内直接调用网页翻译接口并批量输出译文；Token 过期或网站更新后，需要重新抓包。
